This project was developed as the capstone project of my bachelor's degree; combining embedded systems, computer vision and machine learning and applied to a real-world road safety problem.

Driver fatigue is a leading cause of road accidents worldwide. Long-distance driving, night shifts and irregular sleep cycles progressively impair a driver's alertness often without them realising it until it is too late. While some modern vehicles include driver monitoring systems, these are typically proprietary solutions available only in high-end vehicles and not accessible as affordable aftermarket options.

This project addresses that gap by building a low-cost, edge-based driver monitoring system that can be installed in any vehicle. An infrared camera continuously captures the driver's face and extracts eye and mouth regions. Two convolutional neural networks predict eye closure and yawning in real time and a modified PERCLOS algorithm combines both signals into a drowsiness score. If the score exceeds a threshold, a buzzer alarm alerts the driver immediately.

The system also includes an accelerometer-based accident detection module that automatically saves recent footage when a collision is detected and a companion web portal (Django + React) for distributing model updates and collecting real-world feedback for continuous improvement.

Design constraint

All inference runs on-device with no internet connection required. TensorFlow Lite models optimized for Raspberry Pi 4's constrained compute budget.

Problem Statement

Driver fatigue significantly increases the risk of road accidents. The problem is that drowsiness develops gradually and is difficult for drivers to self-monitor by the time a driver notices they are fatigued, their reaction time and attention have already been compromised. Existing driver monitoring solutions are either integrated into expensive new vehicles or require proprietary hardware. There is no widely accessible, affordable aftermarket system that can be retrofitted into any car to provide real-time drowsiness detection.

Drowsiness Detection Pipeline

The detection pipeline processes each video frame in real time. Dlib's 68-point facial landmark model identifies key facial feature locations, from which the left eye, right eye and mouth regions are cropped and passed to the CNN models. The two model outputs are then combined by the PERCLOS scoring algorithm to compute a continuous drowsiness estimate.

CNN Model 1
Eye-State Detection

Binary classifier predicting whether the driver's eyes are open or closed from the cropped eye region.

CNN Model 2
Yawning Detection

Binary classifier detecting yawning behavior from the cropped mouth region across consecutive frames.

Modified PERCLOS Drowsiness Score
Score = 0.75 × Eye Closure + 0.25 × Yawning
PERCLOS (Percentage of Eye Closure) is a validated fatigue metric that measures the proportion of time eyes are 80%+ closed over a rolling window. This implementation extends the standard PERCLOS with a weighted yawning component thereby reducing false positives that arise from relying on eye closure alone. If the computed score exceeds the predefined threshold, the buzzer alarm is triggered immediately.
Facial landmark extraction

Dlib's 68-point model identifies precise facial feature locations from each frame, enabling consistent region extraction regardless of head pose or slight movement.

  • L Left eye: landmarks 36–41
  • R Right eye: landmarks 42–47
  • M Mouth: landmarks 48–67
Accident detection trigger

The MPU6050 accelerometer continuously monitors vehicle acceleration. If sudden deceleration exceeds 8 m/s², the system saves the most recent frames from its circular buffer to local storage, enabling post-incident review and model improvement.

Challenges & Solutions
Challenge
Real-Time Inference on Edge Hardware
Running two CNN models plus facial landmark detection in real time on a Raspberry Pi 4 required significant optimization to avoid frame drops that would degrade detection reliability.
Solution
Converted models to TensorFlow Lite for hardware-optimized inference, used grayscale preprocessing to reduce input dimensionality and minimized per-frame processing overhead through efficient OpenCV pipelines.
Challenge
Robust Facial Feature Detection
Accurate, consistent extraction of eye and mouth regions was critical, slight misalignment or missed detections would propagate errors directly into the drowsiness score and generate false alarms.
Solution
Used Dlib's 68-point facial landmark model to consistently localize feature regions with sub-pixel accuracy regardless of head pose variation, illumination changes or minor movement.
Challenge
Reducing False Positive Alerts
Using eye closure alone as a drowsiness signal can cause false alarms, since normal blinking, looking away or brief occlusions may trigger alerts and reduce driver confidence in the system.
Solution
Implemented a modified PERCLOS metric that combines eye closure frequency (weighted 0.75) with yawning frequency (weighted 0.25) over a rolling window thereby significantly improving specificity without hurting sensitivity.
Outcome

The system successfully demonstrated real-time detection of driver fatigue using computer vision and edge AI inference running entirely on a low-cost Raspberry Pi 4 with no cloud dependency, making it viable as an affordable aftermarket safety solution for any vehicle.

Key outcomes included reliable detection of eye closure and yawning behavior, real-time alerting through the onboard buzzer, automatic accident footage recording via the accelerometer module and a modular architecture enabling ongoing model improvement through the companion web portal. The project demonstrates how embedded AI systems can improve road safety by monitoring driver attentiveness in real time without requiring expensive OEM hardware.
